# 3C 草案完整最终版

## 1. 背景 Context

项目名称：
`个人消费管理与储蓄规划平台`

目标用户：
`需要记录日常收支、控制消费、制定预算、规划储蓄的人群`

核心问题：
`用户日常消费零散，缺少清晰的收支记录、预算控制和储蓄规划，容易花钱无计划、存钱没目标`

使用场景：
`生活日常`

优先级：
`先完成注册登录、记账导入、基础统计、图表展示等核心功能，再逐步扩展预算管理、储蓄规划和个性化分析服务`

项目定位：
`一个面向个人用户的生活消费管理类 Web App，帮助用户记录收支、管理消费、分析趋势，并在需要时提供预算与储蓄规划支持`

限制条件：
`前期先以网页形式实现，后续再接入服务器、数据库和用户系统；当前设计要兼顾后续扩展性`

是否需要注册登录：
`需要，用户必须注册登录后才能管理自己的数据`

是否支持一键导入账单：
`支持，用户可以上传已有账单进行导入`

是否包含预算管理：
`包含，但属于可选扩展功能，用户可以选择是否设置每日、每月、每年预算`

是否包含储蓄规划：
`包含，但属于可选扩展功能，用户可以选择是否填写储蓄目标`

是否包含个性化服务：
`包含，通过用户支出情况判断消费水平并给出个性化建议`

## 2. 步骤 Command

用户输入：
`用户输入收入、支出、分类、日期、备注等基础信息；也可以选择性输入预算金额、储蓄目标金额和储蓄期限`

输入方式：
`支持手动录入和一键导入账单两种方式`

系统处理：
`系统校验输入内容并保存账单数据；手动录入时使用用户填写的时间，导入账单时使用账单原始记录时间；随后更新统计结果，并在用户填写预算或储蓄目标时生成预算对比、储蓄建议、图表分析和消费评估`

处理逻辑：
`先完成用户身份确认，再处理账单录入或导入。手动录入的收支记录按用户填写时间保存，导入账单的收支记录按账单原始时间保存，不按导入时刻保存。随后系统根据用户已填写内容进行统计分析、预算计算、储蓄规划、消费分析和结果展示`

预算对比：
`当用户设置了每日、每月或每年预算后，系统将实际支出与预算进行对比；若用户未设置预算，则不执行预算提醒`

储蓄规划计算：
`当用户填写了“多久存多少钱”后，系统反推每日、每月或每年建议储蓄金额；若未填写，则不生成储蓄建议`

消费水平判断：
`根据用户支出金额、消费频率、消费分类占比和预算执行情况，判断消费水平并提供针对性建议`

图形化展示内容：
`展示每日、每月、每年的收入变化、支出变化、结余变化、预算执行情况和存款进度`

结果输出：
`输出当前收支情况、预算对比结果、超支提醒、储蓄建议、消费水平判断和图表分析结果`

页面状态：
`需要覆盖空输入、错误、处理中、成功四类状态`

页面流程：
1. `用户打开平台并进入注册/登录页面`
2. `用户完成注册或登录后进入首页仪表盘`
3. `用户通过手动录入或一键导入方式添加账单`
4. `用户可选设置每日、每月、每年预算以及储蓄目标`
5. `系统根据用户已填写内容进行统计分析、预算对比、储蓄规划和消费分析`
6. `用户查看图表、报告、提醒和个性化建议`

## 3. 标准 Criteria

成功状态：
`用户能够成功注册登录、录入或导入账单；若填写了预算或储蓄目标，系统还能生成对应的预算对比和储蓄建议`

失败状态：
`注册失败、登录失败、账单保存失败、导入失败、图表加载失败或分析生成失败`

空输入处理：
`当用户未填写必要信息时，系统应阻止提交并明确提示缺少的内容`

错误处理：
`当输入格式错误、金额非法、导入文件异常或处理失败时，系统应提示具体原因并允许重新操作`

处理中状态：
`在注册、登录、导入、保存、计算和加载图表时，系统应显示处理中状态并防止重复提交`

超支提醒规则：
`当用户已设置预算且今日、本月或本年支出超过对应预算时，系统应给出明确提醒；未设置预算时不触发该提醒`

导入账单验收：
`用户导入账单后，系统能够正确解析账单内容，并将每条收支记录按账单原始时间保存和展示，而不是按导入时间记录`

图表展示验收：
`系统能够正确展示收入、支出、结余、预算执行和存款变化的图形化结果`

个性化建议验收：
`系统能够基于用户已有账单数据输出消费分析；如果用户填写了预算或储蓄目标，则可进一步提供更有针对性的建议`

整体验收标准：
1. `用户能够成功注册并登录，且只能查看和管理自己的数据`
2. `用户能够通过手动录入或一键导入两种方式添加账单`
3. `当用户设置了每日、每月、每年预算后，系统能够将实际支出与预算进行对比，并在超支时提醒`
4. `当用户填写储蓄目标后，系统能够生成每日、每月或每年的储蓄建议`
5. `系统能够图形化展示收支和存款变化，并正确处理空输入、错误、处理中和成功状态`
6. `用户导入账单后，系统展示的每条收支记录时间必须与账单原始时间一致`
